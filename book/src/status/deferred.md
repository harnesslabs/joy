# Roadmap / Deferred Features

Status entries below are intentionally explicit so users know what is not ready yet.

## Deferred (Known, Expected Future Work)

- Workspace-wide lockfile strategy (today: per-member lockfiles)
- Broader registry protocols / sparse index options beyond the current git-backed registry mode
- Registry alias package support (registry name != canonical source package ID)
- Deeper package feature/variant support and non-binary target kinds
- Full-screen TUI mode (the current roadmap focuses on a polished line-oriented CLI)

## Planned Documentation Notes

This book may describe roadmap intent in select chapters. Planned sections are always labeled and should not be interpreted as shipped behavior.
